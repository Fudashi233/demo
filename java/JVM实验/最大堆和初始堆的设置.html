源码

package test;

import java.lang.management.ManagementFactory;

public class Test {

    private static int count = 0;

    public static void main(String[] args) {

        System.out.printf("max memory:%d\n",getMaxMemory());
        System.out.printf("free memory:%d\n",getFreeMemory());
        System.out.printf("total memory:%d\n",getTotalMemory());

        /* 分配1M空间 */
        System.out.println("分配1M空间");
        byte[] room = new byte[1*1024*1024];

        System.out.printf("max memory:%d\n",getMaxMemory());
        System.out.printf("free memory:%d\n",getFreeMemory());
        System.out.printf("total memory:%d\n",getTotalMemory());

        /* 分配4M空 */
        System.out.println("分配4M空间");
        room = new byte[1*1024*1024];

        System.out.printf("max memory:%d\n",getMaxMemory());
        System.out.printf("free memory:%d\n",getFreeMemory());
        System.out.printf("total memory:%d\n",getTotalMemory());
    }

    /**
     * 打印虚拟机配置参数
     */
    private static void printCommand() {
        System.out.printf("VM arguments:%s\n",ManagementFactory.getRuntimeMXBean().getInputArguments());
    }


    private static long getMaxMemory() {
        return Runtime.getRuntime().maxMemory();
    }

    private static long getFreeMemory() {
        return Runtime.getRuntime().freeMemory();
    }

    private static long getTotalMemory() {
        return Runtime.getRuntime().totalMemory();
    }
}

总结

  1. -XX:+PrintCommandLineFlags会打印传递给虚拟的，显式或隐式参数，与之配对的还有
    -XX:+PrintVMOptions
-XX:+PrintCommandLineFlags
-XX:+PrintFlagsFinal 打印出JVM运行时的参数
-XX:+PrintFlagsInitial 打印出JVM的默认值
  2. -XX:InitialHeapSize=1048576指定的值就是110241024,与VM arguments指定的一样 -XX:MaxHeapSize=20971520指定的值也与VM arguments指定的值一样
    -Xmx20M
-Xms1M

  3. 根据GC日志，堆的最大大小（max memory）并不与VM agruments指定的一直，这是因为 GC运行本身需要占据一定的空间，而且堆的字节对齐也会浪费一定空间，所以VM arguments指定的堆大小仅仅是一个近似值 GC日志中打印的堆大小是真实值。总之，实际可用堆大小与-Xmx指定的值之间有一定的偏差。

  4. 当堆大小不满足应用程序所申请的空间时，其会自行扩大（受到-Xmx的限制）或者触发GC

  5. 调优技巧：准确估计堆运行时的最大值，然后将-Xmx和-Xms都设置为这个值(略大)，从而减少垃圾 回收次数，提高性能
