TLAB

简介

TLAB（thread local allocation buffer），叫做线程本地分配，是一种JVM优化技术。 Java是一种多线程语言，堆由各个线程共享，当某个线程向堆申请空间时，为了线程安全必然需要对这一操作进行同步， 而同步就会因为加锁、释放锁、阻塞等诸多原因而消耗资源。 所以，JVM使用了TLAB这种线程专属区间（通过为每一个线程在eden区分配一块独立的空间）来避免这一资源损耗，从而提高效率。 使用-XX:+UseTLAB启用这一优化策略。

package test;

TLABRefillWasteFraction

一般TLAB不会太大，假设一个100K的TLAB，剩余20K。现在其所属线程要求 分配一个30K的内存空间，这时就有两种可能。一，废弃该TLAB，重新构造一个 TLAB，这样那剩余的20K就浪费了。二，直接向堆内存申请空间，保留该TLAB。 这两种可能是由一个-XX:TLABRefillWasteFraction虚拟机参数控制的，默认值为 64,另外，引入一个叫refill_waste的变量。
refill_waste=TLAB的大小/TLABRefillWasteFraction
假设TLABRefillWasteFraction为64，TLAB为100K时，refill_waste的大小就是 1.54K（100/64=1.54）。当TLAB的剩余空间不满足线程要求分配的空间时， 当要求分配的空间大于refill_waste就直接在堆上分配，小于就重写构建一个TLAB来分配空开。

默认情况下，refill_waste会不断调整，时系统的运行状态达到最优。 使用-XX:-ResizeTLAB禁用这一调整，使用-XX:TLABSize指定TLAB的大小。
