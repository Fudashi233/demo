finalize()方法影响GC

简介

Object类有个finalize()方法，它借鉴了C++中析构函数的思想，当对象被GC之前会调用该函数。 但这是一个很鸡肋的方法。 1.引用外泄造成对象复活 2.由于GC调用时间不定，finalize()方法的调用时间也就不确定，甚至由于GC不调用，finalize()也不会被调用 3.糟糕的finalize会影响GC的性能从而影响应用程序的性能

源码

package test;

import java.util.HashMap;
import java.util.Map;

public class Test {

    public static void main(String[] args) throws InterruptedException {

        for(int i=0;i<50000;i++) {
            Item item = new Item();
        }
    }

    private static class Item {

        private byte[] b = new byte[512];
        @Override
        public void finalize() {
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}
