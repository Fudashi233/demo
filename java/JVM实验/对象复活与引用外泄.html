对象复活与引用外泄

源码

package test;

import java.nio.ByteBuffer;
import java.util.Vector;

public class Test {

    private static Test obj;

    @Override
    protected void finalize() throws Throwable {
        System.out.println("invoke finalize()");
        super.finalize();
        obj = this;
    }
    public static void main(String[] args) throws InterruptedException {

        obj = new Test();
        System.out.println(obj);
        obj = null;
        System.gc();
        Thread.sleep(1000);
        if(obj==null) {
            System.out.println("the obj is null");
        } else {
            System.out.println(obj);
        }
        obj = null;
        System.gc();
        Thread.sleep(1000);
        if(obj==null) {
            System.out.println("the obj is null");
        } else {
            System.out.println(obj);
        }
    }
}

在finalize()方法的调用过程中，由于obj = this; 这条语句的存在，导致obj对象任然可达（可触及），这就是所谓的对象复活。 java对象被真正清除前会调用其finalize()方法，如果对象在方法调用过程中复活了 则对象无法被GC，如果没有复活，那么就会转为不可达状态，从而被GC清除。 finalze()方法仅会被调用一次

为什么finalize()方法是一个糟糕的方法

  1. 引用外泄造成对象复活
  2. 调用时间不明确，在不发生GC的极端情况下，finalize()甚至不会被调用
